Recursos de Segurança AWS

Serviços e Ferramentas para Proteger sua Infraestrutura na Nuvem

🎯 Visão Geral dos Recursos de Segurança

A AWS oferece um portfólio abrangente e profundo de serviços de segurança nativos da nuvem, projetados para proteger dados, aplicações e infraestrutura. Esses recursos cobrem todas as camadas de segurança - desde identidade e controle de acesso até detecção de ameaças, proteção de dados, segurança de rede e conformidade regulatória.

Diferente de soluções on-premises tradicionais, os serviços de segurança AWS são profundamente integrados com a plataforma, escaláveis automaticamente, pagos conforme uso e frequentemente utilizam machine learning para detecção avançada de ameaças. Esta integração nativa permite implementar segurança de forma mais eficiente e eficaz do que seria possível construir do zero.

Princípio de Defesa em Profundidade: A AWS segue o modelo de defesa em camadas múltiplas. Nenhum controle único é suficiente - você deve implementar controles preventivos, detectivos e responsivos em todas as camadas. Este guia explora os principais recursos de segurança AWS e como utilizá-los efetivamente para construir uma postura de segurança robusta.

👤 Identidade e Gerenciamento de Acesso

AWS Identity and Access Management (IAM)

O QUE É: Serviço fundamental que controla quem pode acessar o quê na sua conta AWS. Gerencia usuários, grupos, roles e suas permissões.

COMPONENTES PRINCIPAIS: Users (identidades humanas ou aplicações), Groups (coleções de usuários), Roles (identidades assumidas temporariamente), Policies (documentos JSON que definem permissões).

RECURSOS CHAVE: Multi-Factor Authentication (MFA), Password Policies, Access Keys para acesso programático, Service Control Policies (SCPs) para Organizations, Permission Boundaries para limitar permissões máximas.

BEST PRACTICES: Nunca use conta root para operações diárias, sempre habilite MFA para usuários privilegiados, siga princípio do menor privilégio, use roles para aplicações EC2/Lambda em vez de access keys, rotacione credenciais regularmente, revise permissões com IAM Access Analyzer.

AWS IAM Identity Center (AWS SSO)

O QUE É: Serviço de Single Sign-On que permite gerenciar acesso a múltiplas contas AWS e aplicações de negócio de um local central.

FUNCIONALIDADES: Login único para todas as contas AWS, integração com provedores de identidade externos (Azure AD, Okta, Google Workspace), gerenciamento centralizado de usuários e grupos, assignment de permissões baseado em grupos.

BENEFÍCIOS: Elimina necessidade de usuários IAM em cada conta, facilita onboarding/offboarding, permite usar identidades corporativas existentes, reduz fadiga de senhas, logs centralizados de acesso.

CASOS DE USO: Organizações com múltiplas contas AWS, empresas que já usam IdP corporativo, ambientes que requerem auditoria centralizada de acesso, equipes que precisam acesso temporário a contas específicas.

Amazon Cognito

O QUE É: Serviço de autenticação, autorização e gerenciamento de usuários para aplicações web e mobile.

COMPONENTES: User Pools (diretórios de usuários com autenticação), Identity Pools (credenciais AWS temporárias para usuários), integração com provedores sociais e corporativos (Google, Facebook, SAML).

RECURSOS: MFA adaptativo, proteção contra bots, customização de fluxos de autenticação, tokens JWT, recuperação de senha, verificação de email/telefone, triggers Lambda para lógica customizada.

QUANDO USAR: Aplicações que precisam gerenciar usuários finais (clientes), mobile apps que requerem autenticação, aplicações web com login social, cenários B2C onde você precisa controle completo sobre experiência de autenticação.

AWS Directory Service

O QUE É: Serviços de diretório gerenciados que permitem usar Microsoft Active Directory na AWS ou conectar recursos AWS ao Active Directory on-premises.

OPÇÕES: AWS Managed Microsoft AD (AD completo gerenciado pela AWS), AD Connector (proxy para AD on-premises), Simple AD (diretório básico compatível com AD).

FUNCIONALIDADES: Autenticação de usuários para EC2, RDS, WorkSpaces, Group Policies, integração com aplicações que requerem AD, trust relationships com domínios on-premises.

CASOS DE USO: Lift and shift de aplicações Windows que usam AD, ambientes híbridos cloud-on-premises, WorkSpaces e AppStream, RDS SQL Server com autenticação Windows.

AWS IAM Access Analyzer

O QUE É: Ferramenta que analisa políticas de recursos e identifica recursos compartilhados com entidades externas à sua conta ou organização.

O QUE ANALISA: S3 buckets, IAM roles, KMS keys, Lambda functions, SQS queues, Secrets Manager secrets - identifica se estão acessíveis externamente.

RECURSOS: Findings de acesso externo, policy validation (valida sintaxe JSON), policy generation (gera policies baseadas em CloudTrail logs), unused access analysis (identifica permissões não usadas nos últimos 90 dias).

BENEFÍCIOS: Previne exposição acidental de dados, ajuda implementar least privilege, facilita compliance, detecta configurações inseguras antes de se tornarem problemas.

🛡️ Detecção de Ameaças e Monitoramento

Amazon GuardDuty

O QUE É: Serviço de detecção de ameaças que monitora continuamente atividades maliciosas e comportamento não autorizado usando machine learning.

FONTES DE DADOS: Analisa CloudTrail events, VPC Flow Logs, DNS logs, Kubernetes audit logs, S3 data events, EKS logs, RDS login activity, Lambda network activity.

TIPOS DE DETECÇÃO: Reconhecimento (port scanning, unusual API calls), comprometimento de instâncias (malware, cryptomining, backdoors), comprometimento de contas (credenciais expostas, anomalias de acesso), exfiltração de dados (transferências incomuns).

RECURSOS: Findings com severity levels, integração com Security Hub, EventBridge para automação, threat intelligence feeds da AWS e terceiros, proteção para múltiplas contas via Organizations.

PRICING: Baseado em volume de dados analisados. 30 dias trial gratuito. Geralmente $4-5 por milhão de eventos CloudTrail, $1-1.50 por GB de VPC Flow Logs.

AWS Security Hub

O QUE É: Serviço centralizado de gerenciamento de segurança que agrega, organiza e prioriza findings de múltiplos serviços AWS e ferramentas de parceiros.

INTEGRAÇÕES NATIVAS: GuardDuty, Inspector, Macie, IAM Access Analyzer, Firewall Manager, Systems Manager Patch Manager, Health, Config - todos enviam findings para Security Hub automaticamente.

SECURITY STANDARDS: Verifica automaticamente conformidade contra CIS AWS Foundations Benchmark, AWS Foundational Security Best Practices, PCI-DSS. Cada standard tem dezenas de checks automatizados.

RECURSOS: Insights (agrupamentos automáticos de findings), cross-region aggregation, automated response actions via EventBridge, integração com SIEMs externos (Splunk, Sumo Logic), ASFF (AWS Security Finding Format) padronizado.

BENEFÍCIOS: Dashboard único para postura de segurança, reduz MTTD (mean time to detect), priorização baseada em severidade, facilita compliance continuous, elimina silos de ferramentas de segurança.

Amazon Detective

O QUE É: Serviço de análise e investigação de segurança que facilita análise, investigação e identificação rápida da causa raiz de findings de segurança.

COMO FUNCIONA: Coleta logs automaticamente de VPC Flow Logs, CloudTrail, GuardDuty findings. Usa machine learning e gráficos para construir visualizações interativas de atividades e relacionamentos.

RECURSOS: Finding groups (agrupa findings relacionados), entity timelines (linha do tempo de atividades de usuário/recurso), geolocation analysis, behavior graphs (visualiza padrões normais vs anômalos), integração nativa com GuardDuty e Security Hub.

CASOS DE USO: Investigação de GuardDuty findings, análise forense de incidentes, threat hunting, validação de alertas (true positive vs false positive), compreensão de escopo de comprometimento.

BENEFÍCIOS: Reduz tempo de investigação de horas para minutos, visualizações intuitivas mesmo para não-especialistas, context-aware (mostra atividades relacionadas automaticamente), não requer configuração complexa de logs.

Amazon Macie

O QUE É: Serviço de descoberta e proteção de dados que usa machine learning para descobrir, classificar e proteger dados sensíveis em S3.

O QUE DETECTA: Personally Identifiable Information (PII) - CPF, RG, passaportes, cartões de crédito, endereços. Financial data - números de contas bancárias. Healthcare data - informações médicas. Credentials - API keys, senhas.

FUNCIONALIDADES: Automated discovery jobs (scans periódicos), sensitive data findings (localização exata de dados sensíveis), bucket policies evaluation (identifica buckets públicos ou compartilhados), data classification (categoriza automaticamente dados).

CASOS DE USO: Compliance com GDPR/LGPD (identificar dados pessoais), PCI-DSS (localizar dados de cartões), descoberta de shadow IT (dados armazenados inadequadamente), prevenção de data leaks, auditoria de segurança de dados.

PRICING: Baseado em dados avaliados. $1 por GB para os primeiros 1TB, preço reduzido para volumes maiores. Bucket-level evaluation gratuito.

AWS CloudTrail

O QUE É: Serviço de auditoria e logging que registra todas as chamadas API na sua conta AWS - quem fez o quê, quando, de onde e com qual resultado.

O QUE REGISTRA: Management events (operações de controle - criar/deletar recursos), data events (operações de dados - GetObject S3, Invoke Lambda), insights events (atividades incomuns detectadas automaticamente).

RECURSOS: Multi-region trails (logs de todas as regiões), organization trails (logs de todas as contas), log file validation (integridade criptográfica), encryption com KMS, integração com CloudWatch Logs, EventBridge para alertas em tempo real.

CASOS DE USO: Auditoria de segurança, compliance, troubleshooting operacional, análise forense de incidentes, detecção de uso não autorizado, tracking de mudanças de configuração.

BEST PRACTICES: Ative em TODAS as regiões, habilite log file validation, criptografe com KMS, armazene em bucket S3 dedicado com acesso restrito, configure lifecycle policies para retenção apropriada, monitore para desativação de trails.

Amazon CloudWatch

O QUE É: Serviço de monitoramento e observabilidade para recursos AWS, aplicações e serviços on-premises.

COMPONENTES DE SEGURANÇA: CloudWatch Logs (centraliza logs de aplicações e AWS services), Metrics (métricas de performance e segurança), Alarms (notificações quando thresholds são excedidos), Events/EventBridge (automação baseada em eventos), Insights (queries avançadas em logs).

SECURITY USE CASES: Alertas de atividades suspeitas (uso de root account, falhas de MFA), monitoramento de mudanças (Security Groups, IAM policies), anomaly detection (baseline de comportamento normal), log aggregation (VPC Flow Logs, application logs).

INTEGRAÇÃO: Recebe logs de CloudTrail, VPC Flow Logs, Lambda, RDS, ECS, EKS. Pode acionar Lambda functions, SNS notifications, Systems Manager automations baseado em eventos.

EXEMPLO DE ALARMES: Root account usage, unauthorized API calls, IAM policy changes, Network ACL changes, Security Group changes, failed console logins, disabling of CloudTrail.

🔒 Proteção de Dados e Criptografia

AWS Key Management Service (KMS)

O QUE É: Serviço gerenciado de criação e controle de chaves de criptografia usadas para proteger dados.

TIPOS DE CHAVES: AWS managed keys (gerenciadas pela AWS, gratuitas), Customer managed keys (você controla, rotação, policies), AWS owned keys (usadas internamente pela AWS, invisíveis), CloudHSM keys (armazenadas em HSMs dedicados).

FUNCIONALIDADES: Automatic key rotation (anual para CMKs), key policies (controle granular de quem pode usar chaves), grants (permissões temporárias programáticas), audit com CloudTrail (todos os usos de chaves são logados), multi-region keys (replique chaves entre regiões).

INTEGRAÇÃO: S3, EBS, RDS, DynamoDB, Lambda, Systems Manager Parameter Store, Secrets Manager, SQS, SNS - todos integram nativamente com KMS.

PRICING: $1/mês por CMK, $0.03 por 10.000 requests. AWS managed keys são gratuitas mas você paga pelos requests.

AWS Secrets Manager

O QUE É: Serviço para armazenar, rotacionar e gerenciar credenciais, API keys e outros secrets de forma segura.

RECURSOS: Automatic rotation (integração nativa com RDS, Redshift, DocumentDB), versioning (mantém histórico de secrets), encryption at rest com KMS, cross-account access (compartilhe secrets entre contas), replica secrets (multi-region).

CASOS DE USO: Database credentials, API keys de terceiros, OAuth tokens, SSH keys, certificados privados, connection strings, qualquer dado sensível usado por aplicações.

BENEFÍCIOS vs HARDCODING: Elimina secrets em código/config files, rotação automática sem downtime de aplicação, audit trail completo de acessos, integração com IAM para controle de acesso granular, recuperação de secrets deletados acidentalmente (30 dias).

PRICING: $0.40 por secret por mês, $0.05 por 10.000 API calls. Considerável para muitos secrets, mas benefícios de segurança justificam custo.

AWS Certificate Manager (ACM)

O QUE É: Serviço para provisionar, gerenciar e implantar certificados SSL/TLS públicos e privados.

RECURSOS: Certificados públicos gratuitos (para uso com serviços AWS), renovação automática (você nunca precisa se preocupar com expiração), wildcard certificates, Private CA (autoridade certificadora privada para uso interno).

INTEGRAÇÃO: Elastic Load Balancing, CloudFront, API Gateway, Elastic Beanstalk, App Runner. Deployment automático - você não gerencia arquivos de certificado.

CASOS DE USO: HTTPS para websites, APIs seguras, comunicação interna criptografada, compliance com requisitos de criptografia em trânsito, mutual TLS authentication.

BENEFÍCIOS: Elimina custos de certificados comerciais, não há risco de expiração de certificado causando outage, facilita melhores práticas de segurança, auditoria de uso via CloudTrail.

AWS CloudHSM

O QUE É: Módulos de segurança de hardware (HSMs) dedicados que atendem requisitos rigorosos de compliance para chaves criptográficas.

DIFERENÇA vs KMS: HSMs single-tenant (apenas você usa), você controla chaves completamente, suporta algorithms não disponíveis em KMS, certificações FIPS 140-2 Level 3, requisito para alguns compliance frameworks.

CASOS DE USO: Compliance que requer HSMs dedicados (PCI-DSS para payment processing), offloading SSL/TLS processing, proteção de chaves privadas para PKI, aplicações que requerem PKCS#11, Java JCE ou Microsoft CNG.

CONSIDERAÇÕES: Significativamente mais caro que KMS ($1.50/hora por HSM), você gerencia alta disponibilidade (mínimo 2 HSMs em AZs diferentes), complexidade operacional maior, requer expertise em HSMs.

QUANDO USAR: Apenas se você tem requisitos específicos que KMS não atende - regulamentações, contractual obligations, aplicações existentes que requerem HSMs.

Amazon Macie (Proteção de Dados)

CAPABILITIES DE PROTEÇÃO: Além de discovery, Macie oferece proteção contínua de dados sensíveis em S3.

AUTOMATED MONITORING: Jobs contínuos que monitoram novos objetos em S3, alertas quando dados sensíveis são detectados, tracking de mudanças em bucket policies que afetam segurança.

REMEDIATION: Integração com EventBridge para automação - quando dados sensíveis detectados, pode automaticamente alterar permissões de bucket, mover objetos para storage mais seguro, notificar equipe de segurança.

COMPLIANCE: Findings detalhados para auditorias, relatórios de onde dados regulamentados estão armazenados, evidências para compliance com GDPR, LGPD, CCPA, PCI-DSS, HIPAA.

🌐 Segurança de Rede

AWS WAF (Web Application Firewall)

O QUE É: Firewall de aplicação web que protege contra exploits e bots comuns que afetam disponibilidade, comprometem segurança ou consomem recursos excessivos.

PROTEÇÕES: SQL injection, Cross-Site Scripting (XSS), malicious bots, rate limiting (DDoS layer 7), geo-blocking (bloquear países específicos), IP reputation lists (IPs maliciosos conhecidos).

AWS MANAGED RULES: Rule groups pré-configurados mantidos pela AWS - Core Rule Set (OWASP Top 10), Known Bad Inputs, SQL Database, Linux/Windows/PHP specific, Admin Protection, Amazon IP Reputation List, Anonymous IP List.

CUSTOM RULES: Crie suas próprias regras baseadas em headers, query strings, body, URIs. Use rate-based rules para DDoS protection. CAPTCHA challenges para suspicious requests.

INTEGRAÇÃO: Application Load Balancer, API Gateway, CloudFront, AppSync, Cognito user pools. Deploy em minutos sem mudanças de arquitetura.

PRICING: $5/mês por Web ACL, $1/mês por rule, $0.60 per million requests. Managed rules adicionam custo mas economizam tempo tremendamente.

AWS Shield

O QUE É: Serviço de proteção contra DDoS (Distributed Denial of Service) gerenciado que protege aplicações na AWS.

SHIELD STANDARD: Gratuito, automaticamente incluído. Protege contra ataques DDoS mais comuns de camada 3 e 4 (network/transport). Disponível para CloudFront, Route 53, ELB, Global Accelerator. Detecta e mitigar ataques em segundos.

SHIELD ADVANCED: $3.000/mês, proteção adicional contra ataques maiores e mais sofisticados. Inclui: DDoS Response Team (DRT) 24/7, proteção de custo (créditos para scaling durante ataque), proteção estendida para EC2, visibility avançada, proteção de camada 7.

QUANDO USAR ADVANCED: Aplicações críticas de negócio, alvos prováveis de ataques DDoS, compliance que requer DDoS protection documentada, workloads onde custo de downtime é alto, necessidade de SRT support durante incidentes.

INTEGRAÇÃO: Funciona em conjunto com WAF para proteção em camadas (Shield para volumetric attacks, WAF para application attacks).

AWS Firewall Manager

O QUE É: Serviço de gerenciamento central de firewall rules em múltiplas contas e recursos AWS Organizations.

O QUE GERENCIA: WAF rules, Shield Advanced protections, Security Groups, Network Firewalls, Route 53 Resolver DNS Firewall. Policies aplicadas automaticamente em novos recursos/contas.

RECURSOS: Policy deployment automático, compliance dashboard (veja recursos out-of-compliance), automatic remediation, hierarchical policies (organizational, account, resource level), audit security groups centrally.

CASOS DE USO: Governança de segurança em múltiplas contas, enforcement de baseline de segurança, compliance com políticas corporativas, simplificação de gestão em escala, prevenção de misconfigurations.

BENEFÍCIOS: Elimina necessidade de configurar WAF/Security Groups manualmente em cada conta, garante consistência de políticas, detecta e corrige drift automaticamente, reduz overhead operacional significativamente.

AWS Network Firewall

O QUE É: Firewall de rede stateful gerenciado que permite filtrar tráfego no nível de VPC.

CAPACIDADES: Stateful inspection (mantém contexto de conexões), IDS/IPS (intrusion detection/prevention), domain filtering (bloquear domínios específicos), protocol detection, deep packet inspection, integração com AWS managed threat intelligence.

CASOS DE USO: Segmentar VPCs complexas, proteger conexões VPN/Direct Connect, inspecionar tráfego entre VPCs (Transit Gateway), filtrar tráfego outbound (prevenir data exfiltration), compliance que requer IDS/IPS.

RULE GROUPS: Managed rule groups (AWS e parceiros como Proofpoint, Forcepoint), custom stateful rules (baseadas em Suricata IDS syntax), stateless rules (ACL-style filtering).

PRICING: $0.395/hora por firewall endpoint, $0.065 per GB processed. Considerável, mas elimina necessidade de appliances third-party e sua gestão.

Amazon VPC (Recursos de Segurança)

SECURITY GROUPS: Firewalls stateful no nível de instância. Allow rules apenas (tudo negado por padrão). Avaliam tráfego antes de chegar à instância.

NETWORK ACLs: Firewalls stateless no nível de subnet. Suportam allow e deny rules. Avaliados antes de Security Groups. Úteis para block rules explícitas.

VPC FLOW LOGS: Capturam informações sobre tráfego IP. Enviam para CloudWatch Logs, S3 ou Kinesis Data Firehose. Essenciais para troubleshooting, detecção de anomalias, auditoria.

VPC ENDPOINTS: Conectividade privada para serviços AWS sem internet gateway. Interface Endpoints (ENIs privadas) e Gateway Endpoints (S3, DynamoDB). Reduzem exposição e melhoram segurança.

PRIVATE LINK: Expor serviços para outras VPCs de forma privada. Não requer VPC peering, internet gateway, NAT. Tráfego não sai da rede AWS.

AWS PrivateLink

O QUE É: Tecnologia que permite acesso privado e seguro a serviços através da rede AWS, sem exposição à internet.

ARQUITETURA: Service provider expõe serviço via VPC Endpoint Service. Service consumers criam Interface Endpoints em suas VPCs. Tráfego flui através da rede backbone AWS.

BENEFÍCIOS DE SEGURANÇA: Tráfego nunca atravessa internet pública, não requer VPC peering (evita overlapping CIDRs), isolamento de rede completo, integra com Security Groups para controle granular, reduz surface de ataque.

CASOS DE USO: SaaS offerings (oferecer serviço para clientes securamente), shared services (compartilhar serviços entre contas/VPCs), third-party integrations (conectar a parceiros sem exposição), microservices architecture (comunicação segura entre serviços).

AWS SERVICES VIA PRIVATELINK: 100+ serviços AWS acessíveis via VPC Endpoints - S3, DynamoDB, API Gateway, Systems Manager, Secrets Manager, KMS, ECR e muitos outros.

🔍 Avaliação e Análise de Vulnerabilidades

Amazon Inspector

O QUE É: Serviço automatizado de avaliação de vulnerabilidades e gestão de segurança para EC2, Lambda e container images.

O QUE AVALIA: Software vulnerabilities (CVEs em packages), network reachability (portas acessíveis via internet, outros recursos), Lambda code and configuration issues, container images em ECR (vulnerabilidades em layers).

CONTINUOUS SCANNING: Automaticamente escaneia novos recursos quando criados, re-escaneia quando softwares são atualizados, monitora novos CVEs publicados e re-avalia recursos existentes, integra com Systems Manager para patch management.

RISK SCORING: Inspector score (0-10 baseado em CVSS), contextualized findings (considera reachability - vulnerabilidade em recurso privado é menos crítica), prioritização automática, integração com Security Hub.

REMEDIATION: Guidance específico para cada finding, integração com Systems Manager para patch deployment, EventBridge para automação customizada, export para ticketing systems.

PRICING: Baseado em número de instâncias/Lambda functions/containers avaliados. Primeiros 30 dias gratuitos.

AWS Config

O QUE É: Serviço que registra configurações de recursos AWS e avalia conformidade com regras desejadas.

FUNCIONALIDADES: Configuration history (timeline de mudanças em recursos), configuration snapshots (estado de recursos em momento específico), configuration streams (mudanças em tempo real), relationships (como recursos conectam entre si).

CONFIG RULES: AWS managed rules (150+ regras pré-construídas) e custom rules (Lambda functions). Avaliam se recursos estão em conformidade. Exemplos: encrypted-volumes, s3-bucket-public-read-prohibited, iam-password-policy.

REMEDIATION: Automatic remediation via Systems Manager Automation, manual remediation guidance, custom remediation actions via Lambda.

CONFORMITY PACKS: Collections de Config Rules para compliance frameworks completos (CIS Benchmark, PCI-DSS, HIPAA). Deploy framework inteiro com um click.

CASOS DE USO: Compliance auditing, security analysis, change management, troubleshooting, resource inventory, disaster recovery.

AWS Systems Manager - Patch Manager

O QUE É: Automatiza processo de patching de instâncias EC2 e servidores on-premises com security patches e updates.

RECURSOS: Patch baselines (defina quais patches instalar), maintenance windows (quando executar patching), patch groups (agrupe instâncias por patching schedule), compliance reporting (quais instâncias estão out-of-compliance).

SUPPORTED OS: Amazon Linux, Ubuntu, RHEL, CentOS, SUSE, Windows Server. Pode gerenciar também aplicações como Microsoft SQL Server, Exchange.

INTEGRATION: Inspector identifica vulnerabilidades, Patch Manager aplica patches, Config verifica compliance, tudo reportado em Security Hub.

AUTOMATION: Schedule patching automático, auto-reboot se necessário, rollback em caso de falha, pre/post patching scripts, notifications via SNS.

Amazon ECR Image Scanning

O QUE É: Scanning de vulnerabilidades integrado para container images armazenados em Amazon Elastic Container Registry.

TIPOS: Basic scanning (powered by Clair, gratuito, scan on push), Enhanced scanning (powered by Inspector, continuous scanning, mais detalhes).

FUNCIONALIDADES: Detecta CVEs em packages, identifica severity levels, integra com Security Hub, EventBridge para automação, scan results armazenados para auditoria.

BEST PRACTICES: Scan todas as images antes de deploy, configure pipeline CI/CD para bloquear deploy de images com vulnerabilidades críticas, re-scan periodicamente images em produção, use enhanced scanning para compliance.

📊 Compliance e Governança

AWS Audit Manager

O QUE É: Serviço que automatiza coleta de evidências para facilitar auditorias de compliance.

FRAMEWORKS: Pré-construídos para CIS AWS Foundations Benchmark, PCI-DSS, GDPR, HIPAA, SOC 2, ISO 27001, NIST, FedRAMP. Custom frameworks para requisitos específicos.

FUNCIONAMENTO: Mapeia controles para recursos AWS, coleta evidências automaticamente (configurações, logs, políticas), organiza por controle e período, gera assessment reports para auditores.

EVIDENCE COLLECTION: Automated (CloudTrail, Config, Security Hub), manual (uploads de documentos), evidence folders (organizados por data e controle), immutable storage (evidências não podem ser modificadas).

BENEFÍCIOS: Reduz tempo de preparação para auditoria de semanas para dias, elimina trabalho manual de coleta, prova contínua de compliance, facilita auditorias recorrentes.

AWS Organizations

O QUE É: Serviço de gerenciamento central de múltiplas contas AWS.

SERVICE CONTROL POLICIES (SCPs): Guardrails que definem permissões máximas para contas. Preventive controls - impedem ações antes que ocorram. Aplicados em nível de OU ou conta individual.

SECURITY USE CASES: Proibir desativação de CloudTrail, requerer encryption, restringir regiões AWS permitidas, bloquear exposição pública de S3, prevenir deleção de recursos críticos, enforçar uso de MFA.

RECURSOS: Consolidated billing, centralized CloudTrail, tag policies, backup policies, AI services opt-out policies, hierarchical structure (OUs).

BENEFÍCIOS: Governance em escala, isolation de workloads, separação de ambientes (prod, dev, test), blast radius containment, simplified billing.

AWS Control Tower

O QUE É: Forma mais fácil de setup e governança de ambiente AWS multi-conta seguro e compliant baseado em best practices.

LANDING ZONE: Automated setup de environment multi-conta. Inclui: Log Archive account, Audit account, organizational units pré-configuradas, baseline de segurança.

GUARDRAILS: Preventive (SCPs) e Detective (Config Rules). Mandatory (sempre ligados), Strongly Recommended, Elective. Exemplos: disallow public read access to S3, enable encryption for EBS volumes, detect CloudTrail enabled.

ACCOUNT FACTORY: Automated provisioning de novas contas com baseline. Self-service para equipes. Garante consistência. Configuração padrão de segurança aplicada automaticamente.

DASHBOARD: Visibilidade de compliance status, violations alertadas, drift detection (mudanças fora de Control Tower), integrado com Organizations e Service Catalog.

AWS Trusted Advisor

O QUE É: Ferramenta online que analisa ambiente AWS e fornece recomendações em tempo real para reduzir custos, melhorar performance, segurança, fault tolerance, service limits.

SECURITY CHECKS: S3 bucket permissions, Security Groups com portas irrestritas, IAM use (root account usage, MFA), EBS public snapshots, RDS public snapshots, CloudTrail logging, exposed access keys, IAM password policy.

TIERS: Basic/Developer support (7 checks gratuitos), Business/Enterprise support (todos os checks, programmatic access via API, CloudWatch integration para alertas).

REFRESH: Checks executam automaticamente semanalmente. Business/Enterprise podem refresh manualmente ou via API. Integra com EventBridge para automação de remediação.

🚨 Resposta a Incidentes

Amazon EventBridge (CloudWatch Events)

O QUE É: Barramento de eventos serverless que conecta dados de aplicações e serviços AWS.

SECURITY AUTOMATION: Responda automaticamente a eventos de segurança. GuardDuty finding → Lambda function quarantines instance. Config violation → SNS notification + remediation action. IAM policy change → log to S3 + notify security team.

EVENT PATTERNS: Filtre eventos específicos, routing baseado em conteúdo, transforms de dados, múltiplos targets por evento.

TARGETS: Lambda functions, Step Functions, SNS/SQS, Systems Manager Automation, EC2 actions, ECS tasks, Kinesis streams, API destinations (webhooks).

CASOS DE USO: Automated incident response, security orchestration, compliance automation, security event aggregation, alerting workflows.

AWS Systems Manager - Session Manager

O QUE É: Acesso seguro a instâncias EC2 sem necessidade de bastion hosts, SSH keys ou portas abertas.

BENEFÍCIOS DE SEGURANÇA: Elimina necessidade de Security Group rules para SSH/RDP, não requer public IPs, não requer key management, centraliza acesso via IAM, logs completos de sessões em CloudTrail e S3.

RECURSOS: Browser-based shell, port forwarding, document automation, audit trail completo, session recording, restrição de comandos permitidos.

COMPLIANCE: Atende requisitos de auditoria de acesso, provê non-repudiation, facilita investigações forenses, suporta break-glass scenarios com approval workflow.

AWS Step Functions

SECURITY USE CASE: Orquestre workflows complexos de resposta a incidentes de forma visual e auditable.

EXEMPLO DE WORKFLOW: GuardDuty detecta compromised instance → Step Function iniciado → Snapshot EBS volume (forense) → Isola instance (change Security Group) → Notify security team → Create ticket → Run Inspector scan → Generate report.

BENEFÍCIOS: Visual workflow (fácil entender e modificar), error handling robusto, retry logic, parallel execution, audit trail completo, integração nativa com AWS services.

INTEGRAÇÃO: Lambda, ECS, Batch, DynamoDB, SNS, SQS, API Gateway, todos os serviços AWS via SDK integration.

AWS Backup

SECURITY PERSPECTIVE: Backups são componente crítico de incident response e disaster recovery.

RECURSOS: Centralized backup policies, automated backup schedules, lifecycle management (transition para cold storage), cross-region backup copy, cross-account backup (backup em conta separada para proteção adicional).

BACKUP VAULT LOCK: Torna backups imutáveis. Nem root account pode deletar durante retention period. Compliance WORM (Write Once Read Many). Proteção contra ransomware.

SUPPORTED SERVICES: EC2, EBS, RDS, Aurora, DynamoDB, EFS, FSx, Storage Gateway, DocumentDB, Neptune, S3.

RESTORE TESTING: Automated restore testing para validar que backups funcionam. Critical - backups não testados são backups que não funcionam.

🔐 Proteção de Aplicações

AWS App Runner

SECURITY FEATURES: Serviço fully managed para deploy de aplicações com segurança built-in.

RECURSOS: Automatic HTTPS, managed certificates via ACM, VPC integration (acesso a recursos privados), IAM roles para aplicação, secrets integration com Secrets Manager e Parameter Store.

BENEFÍCIOS: Elimina necessidade de gerenciar infraestrutura de segurança, best practices aplicadas automaticamente, patches e updates gerenciados pela AWS, compliance easier.

AWS Lambda (Security Features)

EXECUTION ROLE: IAM role que define permissões da function. Princípio do menor privilégio. Sem chaves hardcoded.

VPC INTEGRATION: Lambda pode executar dentro de VPC para acessar recursos privados. Hyperplane ENIs (não consome IPs da subnet).

ENVIRONMENT VARIABLES: Encryption at rest com KMS. Pode armazenar configurações sensíveis. Melhor prática: use Secrets Manager para secrets realmente sensíveis.

CODE SIGNING: Garante que apenas código aprovado e assinado executa. Previne deployment de código não autorizado. Integra com AWS Signer.

LAYERS SECURITY: Controle quem pode usar suas layers. Compartilhe layers entre contas de forma controlada. Scan layers com Inspector.

Amazon API Gateway

AUTHENTICATION: IAM authentication, Cognito User Pools, Lambda authorizers (custom auth logic), API keys (rate limiting, identificação básica).

AUTHORIZATION: IAM policies (quem pode invocar API), resource policies (quem pode acessar API Gateway), Lambda authorizers (custom authorization logic), Cognito groups.

SECURITY FEATURES: HTTPS only (certificados via ACM), throttling (previne DDoS), WAF integration, CloudWatch logging, AWS X-Ray tracing, VPC Link (privado), mutual TLS authentication.

DATA PROTECTION: Request/response validation, encryption em trânsito, client certificate authentication, API keys rotation, usage plans para rate limiting.

AWS Amplify

SECURITY BUILT-IN: Framework para desenvolvimento de aplicações com segurança integrada.

AUTHENTICATION: Cognito integration out-of-the-box, social sign-in, MFA, password policies, account recovery.

AUTHORIZATION: Fine-grained access control via Cognito groups e IAM, API authorization rules, GraphQL field-level authorization.

HOSTING: HTTPS by default via CloudFront, DDoS protection via Shield, automatic certificate management, branch-based deployments (isolamento de ambientes).

AWS IoT Device Defender

O QUE É: Serviço de segurança para gerenciar segurança de frotas IoT.

AUDIT: Avalia policies e configurações IoT contra best practices de segurança. Identifica gaps de segurança, certificados expirados, policies permissivas.

DETECT: Monitora comportamento de dispositivos para detectar anomalias. Machine learning identifica desvios de baseline. Alertas de atividades suspeitas.

MITIGATE: Actions automatizadas quando ameaças detectadas. Quarantine device, update certificate, notify security team, trigger investigation workflow.

Amazon WorkSpaces / AppStream 2.0

DATA PROTECTION: Data at rest encryption, in-transit encryption, no data stored on user devices, ephemeral sessions, integration com AD para RBAC.

COMPLIANCE: HIPAA eligible, PCI-DSS compliant infrastructure, SOC reports disponíveis, facilita BYOD policies securamente.

🎨 Melhores Práticas de Arquitetura

Defense in Depth

Implemente múltiplas camadas de segurança. Se uma falhar, outras protegem. Network layer (Security Groups, NACLs), Application layer (WAF, Shield), Data layer (encryption, access controls), Identity layer (IAM, MFA), Detection layer (GuardDuty, Config), Response layer (EventBridge automation, incident response procedures).

Least Privilege

Conceda apenas permissões mínimas necessárias. Use IAM Access Analyzer para identificar permissões não utilizadas. Comece com nenhuma permissão, adicione conforme necessário. Revise permissões regularmente. Use time-based permissions quando possível (temporary elevated access). Service Control Policies como guardrails organizacionais.

Security Automation

Automatize detecção e resposta. EventBridge para orquestração, Lambda para remediações, Systems Manager para patch management, Config para compliance enforcement. Reduz MTTR (mean time to respond), elimina erros humanos, escala eficientemente, libera equipe para trabalho estratégico.

Logging e Monitoring

Log tudo, sempre. CloudTrail para API calls, VPC Flow Logs para network traffic, application logs para CloudWatch Logs, access logs para S3/ALB/CloudFront. Retenção apropriada conforme compliance. Alertas para atividades suspeitas. Análise regular de logs. Integration com SIEM se necessário.

Encryption Everywhere

Dados em repouso e em trânsito devem ser criptografados. Use KMS CMKs para controle total. TLS 1.2+ para comunicação. Rotação automática de chaves. Certificate management via ACM. Validate encryption com Config rules. Secrets no Secrets Manager, não hardcoded.

Network Isolation

Use VPCs para isolamento. Subnets privadas para workloads. Public subnets apenas para load balancers e NAT. VPC Endpoints para serviços AWS (tráfego não sai da rede AWS). Security Groups restritivos. Network Firewall para inspection avançada. Transit Gateway para conectividade complexa.

Incident Response Readiness

Tenha plano antes que incidente ocorra. Playbooks documentados para cenários comuns. Automated response onde possível. Forensic readiness (snapshots automáticos, log retention). Teste plano regularmente (tabletop exercises). Clear roles e responsabilidades. Contatos de emergência atualizados.

Regular Security Assessments

Segurança é processo contínuo, não one-time setup. Reviews mensais de Security Hub findings. Quarterly access reviews. Vulnerability scans com Inspector. Penetration testing (notifique AWS). Compliance assessments. Gap analysis contra frameworks. Update procedures conforme ameaças evoluem.

💰 Considerações de Custo

Serviços Gratuitos

SEMPRE GRATUITOS: IAM (gerenciamento de identidade), CloudTrail (primeira trail de management events gratuita), AWS Shield Standard (proteção DDoS básica), ACM (certificados SSL para uso com AWS services).

FREE TIERS: GuardDuty (30 dias), Security Hub (30 dias), Inspector (15 dias EC2/90 dias Lambda), Macie (30 dias), Config (primeiros 2 rules gratuitos).

Otimização de Custos

PRIORIZE: Implemente serviços críticos primeiro (CloudTrail, GuardDuty, Config básico). Use managed rules quando possível (menos custo que custom). Lifecycle policies para logs (transition para Glacier). Filtre logs desnecessários.

ROI: Custos de segurança são investimento. Breach custa muito mais. Automação reduz headcount necessário. Compliance evita multas. Calculate total cost of ownership incluindo operational overhead.

Recursos por Tamanho de Organização

STARTUP/PEQUENO: CloudTrail, IAM + MFA, Security Groups bem configurados, Config básico, GuardDuty, Security Hub free tier, AWS Backup. Total: ~$50-200/mês.

MÉDIO: Adicione WAF, Secrets Manager, Inspector, Macie (se dados sensíveis), enhanced monitoring. Total: ~$500-2000/mês.

ENTERPRISE: Shield Advanced, Network Firewall, CloudHSM (se necessário), Audit Manager, Control Tower, multi-account strategy. Total: $5000+/mês.

⚠️ Armadilhas Comuns

❌ Não Habilitar Serviços de Detecção

ERRO: "Custos de GuardDuty/Security Hub são altos, vou economizar."
REALIDADE: Custo de um breach é 100x maior. Detecção precoce é crítica. Free tiers permitem avaliar valor. Automação reduz necessidade de headcount.

❌ Logging Insuficiente

ERRO: CloudTrail só em algumas regiões, VPC Flow Logs desabilitados para "economizar".
REALIDADE: Sem logs, investigação de incidentes é impossível. Compliance requer logs. Custo de storage é mínimo comparado a valor. Use lifecycle policies para otimizar.

❌ Permissões Excessivas

ERRO: "Vou dar AdministratorAccess por enquanto, depois ajusto."
REALIDADE: "Depois" nunca chega. Permissões excessivas são exploradas. Difícil reduzir depois (quebra coisas). Use Access Analyzer desde início.

❌ Não Testar Backups

ERRO: "Backups configurados, check!"
REALIDADE: Backups não testados não funcionam quando precisa. Teste restore procedures regularmente. Valide RPO/RTO. Backup sem teste é falsa segurança.

❌ Segurança Como Afterthought

ERRO: "Vamos lançar rápido, adicionamos segurança depois."
REALIDADE: Security by design é mais fácil e barato. Retrofit de segurança é caro e complexo. Technical debt de segurança tem juros altos. Shift-left sempre.

❌ Ignorar Alertas

ERRO: "Muitos false positives, vou desabilitar alertas."
REALIDADE: Tune alertas, não ignore. False positives indicam configuração inadequada. Alert fatigue é real mas solução é ajustar, não desabilitar. Pode perder real threats.

🚀 Roadmap de Maturidade

Nível 1 - Fundação (0-3 meses)

IAM setup correto, CloudTrail habilitado, Security Groups básicos, backup configurado, billing alerts, MFA obrigatório, password policies. Foco: prevenir low-hanging fruit attacks.

Nível 2 - Detecção (3-6 meses)

GuardDuty, Security Hub, Config, Inspector, VPC Flow Logs, CloudWatch Alarms. Automation básica via EventBridge. Foco: detectar ameaças rapidamente, compliance básico.

Nível 3 - Resposta (6-12 meses)

Detective, automated remediation, incident response procedures testados, multi-account strategy, WAF, Secrets Manager. Foco: responder a incidentes eficientemente, reduzir blast radius.

Nível 4 - Proativo (12+ meses)

Threat hunting, red team exercises, advanced automation, Macie para data discovery, Network Firewall, continuous compliance, security metrics e KPIs. Foco: prevenir ataques sofisticados, compliance contínuo.

Nível 5 - Otimização (Ongoing)

Machine learning customizado, integration com SIEM enterprise, zero trust architecture, security as code completamente, automated compliance evidence collection. Foco: excelência operacional, inovação contínua.

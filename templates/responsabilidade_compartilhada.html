Modelo de Responsabilidade Compartilhada AWS

Segurança "DA" Nuvem vs Segurança "NA" Nuvem

🎯 O Conceito Fundamental

O Modelo de Responsabilidade Compartilhada da AWS estabelece papéis definidos para a AWS e para o cliente, garantindo a segurança e conformidade dos serviços utilizados na nuvem. Este modelo ajuda a reduzir os encargos operacionais do cliente, pois a AWS opera, gerencia e controla os componentes fundamentais do sistema.

A distinção principal neste modelo é feita entre a segurança "DA" nuvem e a segurança "NA" nuvem.

Responsabilidade da AWS: Segurança "DA" Nuvem

A AWS é responsável pela proteção da infraestrutura global que executa todos os serviços oferecidos na Nuvem AWS. A AWS cuida de tudo aquilo que mantém a nuvem de pé rodando.

Inclui:
• Infraestrutura Física e Instalações
• Hardware (servidores, armazenamento)
• Rede (componentes físicos, roteadores)
• Software de Infraestrutura (hypervisor)
• Controles Heredados

Responsabilidade do Cliente: Segurança "NA" Nuvem

O cliente é responsável por proteger os recursos e dados que utiliza na nuvem. Em outras palavras, o cliente é responsável por usar a nuvem de forma segura.

Inclui:
• Dados do Cliente
• IAM (usuários, permissões, MFA)
• Sistema Operacional Convidado
• Aplicações
• Configuração de Segurança de Rede
• Criptografia

⚖️ Controles Compartilhados

Existem controles que são de responsabilidade compartilhada entre AWS e cliente, mas em contextos diferentes:

Controle Responsabilidade da AWS Responsabilidade do Cliente
Gerenciamento de Patches Aplicação de patches na infraestrutura subjacente Aplicação de patches no SO convidado (EC2) e aplicativos
Gerenciamento de Configuração Manter configuração dos dispositivos de infraestrutura Configuração de bancos de dados, aplicativos e SO convidado
Conhecimento e Treinamento Treinamento de funcionários da AWS Treinamento dos próprios funcionários

🏗️ Variação por Tipo de Serviço (IaaS, PaaS, SaaS)

A quantidade de responsabilidade do cliente varia significativamente dependendo do serviço da AWS selecionado. Quanto mais abstraído o serviço, mais responsabilidade de gerenciamento é transferida para a AWS.

IaaS - Infrastructure as a Service (Amazon EC2)

CLIENTE TEM O MAIOR CONTROLE

O cliente é responsável pelo SO convidado, atualizações, aplicações, dados, IAM e security group. A AWS é responsável pela infraestrutura global (hardware, rede, virtualização).

Exemplo: Para uma instância EC2, o cliente deve manter o Sistema Operacional atualizado com todos os patches de segurança necessários, bem como configurar o security group.

PaaS - Platform as a Service (AWS Lambda, Amazon RDS)

RESPONSABILIDADE INTERMEDIÁRIA

A AWS cuida da infraestrutura e, geralmente, do SO subjacente. No Lambda, o cliente gerencia o código das funções. Em RDS, a responsabilidade pelo SO e configuração de rede é transferida para a AWS, mas o cliente ainda é responsável por dados, IAM e configuração do firewall.

SaaS/Abstrato - Software as a Service (Amazon S3, DynamoDB)

AWS ASSUME A MAIOR PARTE DA RESPONSABILIDADE

A AWS opera a infraestrutura, o SO e as plataformas. Mais segurança é transferida para a AWS, incluindo proteção de dados em repouso e proteção de tráfego de rede para dados em trânsito. O cliente se concentra no gerenciamento de dados, classificação, uso de ferramentas IAM e configuração do serviço.

Responsabilidade IaaS (EC2) PaaS (RDS, Lambda) SaaS (S3, DynamoDB)
Infraestrutura Global AWS AWS AWS
SO Convidado & Rede Cliente AWS AWS
Plataforma/Aplicações Cliente AWS AWS

📋 Classificações Formais de Controles

A. Controles Heredados (AWS Responsável)

Controles que o cliente herda completamente da AWS. O cliente não precisa se preocupar em implementá-los ou verificá-los.

Exemplos:
• Controles Físicos e Ambientais
• Segurança física das instalações e data centers
• Gestão de geradores, energia e ar-condicionado

B. Controles Compartilhados (AWS e Cliente)

Aplicam-se tanto à camada de infraestrutura quanto às camadas do cliente, mas em contextos distintos.

• Gerenciamento de Patches: AWS para infraestrutura, Cliente para SO/apps
• Gerenciamento de Configuração: AWS para dispositivos, Cliente para recursos
• Conhecimento e Treinamento: Cada um treina seus próprios funcionários

C. Controles Específicos do Cliente

São controles de responsabilidade exclusiva do cliente com base na aplicação que ele implanta nos serviços.

Exemplo: Proteção ou zona de segurança de serviços e comunicação, que pode exigir que o cliente roteie dados para ambientes de segurança específicos.

⚠️ Desafios e Práticas Recomendadas

O Cliente Causa a Maioria das Falhas

É essencial que as organizações entendam o seu papel na segurança, pois a falta de controles adequados é frequentemente citada como a principal causa de falhas de segurança na nuvem.

ESTATÍSTICA IMPORTANTE: Prevê-se que, até 2025, 99% das falhas de segurança na nuvem serão culpa do cliente.

O Perigo das Configurações Padrão

As configurações padrão da AWS podem fornecer um nível básico de proteção, mas são provavelmente insuficientes para os requisitos específicos de segurança e conformidade da sua organização.

Você é responsável por garantir que os controles de segurança apropriados estejam ativos. Por exemplo, se você não habilitar MFA ou configurar credenciais de usuário inadequadas, você será responsável por qualquer violação de dados resultante.

Dicas para Cumprir Suas Responsabilidades

1. NÃO ASSUMA COBERTURA TOTAL: Não presuma que todos os aspectos da segurança são tratados exclusivamente pela AWS.

2. CONFIGURAÇÕES CORRETAS: As configurações padrão podem ser insuficientes para seus requisitos específicos.

3. SEGURANÇA PROATIVA: Tome medidas proativas como monitoramento, patching e avaliações de segurança regulares.

4. IAM ADEQUADO: Implemente práticas adequadas de IAM, incluindo MFA e gerenciamento de credenciais.

Responsabilidades Detalhadas do Cliente

DADOS DO CLIENTE: O cliente é o único responsável por gerenciar, armazenar e controlar o acesso, incluindo escolha das opções de criptografia e gestão das chaves.

GERENCIAMENTO DE ACESSO (IAM): Implementação adequada de práticas de IAM, incluindo criação de usuários/funções/permissões e configuração de MFA.

CONFIGURAÇÃO DE REDE E FIREWALL: Configuração correta de regras de Security Group, Network ACLs e VPCs para controlar o fluxo de tráfego.

🎓 Pontos-Chave para a Certificação

Conceitos Essenciais para o Exame

✅ SEGURANÇA "DA" NUVEM: AWS é responsável pela infraestrutura que mantém a nuvem funcionando (hardware, rede, virtualização).

✅ SEGURANÇA "NA" NUVEM: Cliente é responsável por usar a nuvem de forma segura (dados, IAM, SO, aplicações).

✅ IaaS (EC2): Cliente tem maior responsabilidade - gerencia SO, patches, aplicações, dados.

✅ PaaS (RDS, Lambda): AWS gerencia SO e plataforma, cliente gerencia dados e código.

✅ SaaS (S3, DynamoDB): AWS assume mais responsabilidade, cliente foca em dados e configuração.

✅ CONTROLES COMPARTILHADOS: Patches, configuração e treinamento são compartilhados em contextos diferentes.

✅ 99% DAS FALHAS: São causadas por configurações inadequadas do cliente, não pela AWS.

✅ MFA E IAM: Cliente DEVE habilitar e configurar corretamente as ferramentas de segurança da AWS.

✅ REGRA DE OURO: AWS cuida do que mantém a nuvem rodando, Cliente cuida do que coloca NA nuvem.

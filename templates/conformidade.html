Conformidade de Segurança AWS

Frameworks, Padrões e Certificações para Compliance na Nuvem

📋 O Que É Conformidade na AWS?

Conformidade (compliance) refere-se ao cumprimento de padrões, regulamentações e frameworks de segurança estabelecidos por organizações reguladoras, governos e indústrias. Na AWS, conformidade significa garantir que sua infraestrutura e operações atendam aos requisitos específicos do seu setor e localização geográfica.

A AWS mantém um amplo programa de conformidade que abrange certificações globais, regionais e específicas da indústria. Compreender e implementar conformidade não é apenas sobre evitar multas - é sobre proteger dados, construir confiança com clientes e estabelecer práticas operacionais sólidas que reduzem riscos de segurança.

Modelo de Responsabilidade Compartilhada: A AWS é responsável pela conformidade DA nuvem (infraestrutura física, rede, virtualização). Você é responsável pela conformidade NA nuvem (dados, aplicações, configurações, controles de acesso). Este guia foca em sua responsabilidade - como garantir conformidade das suas cargas de trabalho na AWS.

🌍 Principais Frameworks e Regulamentações

CIS AWS Foundations Benchmark

O QUE É: Conjunto de melhores práticas de segurança desenvolvido pelo Center for Internet Security, reconhecido globalmente como padrão baseline para segurança AWS.

ESCOPO: Cobre configurações fundamentais de IAM, logging, monitoramento, networking. Não específico de indústria - aplicável a qualquer organização.

IMPLEMENTAÇÃO: AWS Security Hub fornece verificação automatizada contra CIS Benchmark. Ative Security Hub e habilite o padrão CIS AWS Foundations Benchmark.

POR QUÊ IMPORTANTE: Base sólida de segurança. Muitas organizações exigem conformidade CIS como requisito mínimo. Frequentemente ponto de partida para auditorias.

GDPR (General Data Protection Regulation)

O QUE É: Regulamentação de privacidade da União Europeia que protege dados pessoais de cidadãos da UE, independente de onde empresa está localizada.

APLICA-SE A: Qualquer organização que processa dados de residentes da UE. Multas podem chegar a 4% do faturamento global anual ou €20 milhões.

REQUISITOS CHAVE: Consentimento explícito para coleta de dados, direito ao esquecimento (deletar dados), portabilidade de dados, notificação de breach em 72 horas, designação de DPO (Data Protection Officer).

NA AWS: Use criptografia obrigatória, implemente retenção e deleção automatizada de dados, configure logs de acesso, documente processamento de dados, use regiões EU para dados de cidadãos EU.

LGPD (Lei Geral de Proteção de Dados)

O QUE É: Lei brasileira de proteção de dados inspirada no GDPR, regulamenta tratamento de dados pessoais no Brasil.

APLICA-SE A: Empresas que processam dados de residentes brasileiros, mesmo se sediadas fora do Brasil. Vigência desde setembro de 2020.

REQUISITOS: Semelhantes ao GDPR - consentimento, direitos dos titulares, segurança de dados, notificação de incidentes à ANPD, nomeação de encarregado de dados.

IMPLEMENTAÇÃO AWS: Use região São Paulo (sa-east-1) para dados sensíveis de brasileiros, implemente controles de acesso granulares, mantenha registros de processamento, habilite criptografia end-to-end.

PCI-DSS (Payment Card Industry Data Security Standard)

O QUE É: Padrão de segurança para organizações que processam, armazenam ou transmitem dados de cartões de crédito. Mandatório, não opcional.

NÍVEIS: Baseados em volume de transações anuais. Nível 1 (mais de 6 milhões) requer auditoria anual por QSA. Menores volumes podem fazer auto-avaliação.

12 REQUISITOS: Firewall, não usar defaults, proteger dados armazenados, criptografar transmissão, antivírus, sistemas seguros, restringir acesso, IDs únicos, restringir acesso físico, monitorar, testar, política de segurança.

AWS: Nunca armazene CVV/CVV2, use tokenização, segmente rede com VPCs dedicadas, logs imutáveis, MFA obrigatório, criptografia forte, scans de vulnerabilidade trimestrais.

HIPAA (Health Insurance Portability and Accountability Act)

O QUE É: Lei dos EUA que regula proteção de informações de saúde (PHI - Protected Health Information). Aplicável a healthcare providers, health plans, clearinghouses.

BUSINESS ASSOCIATE AGREEMENT: Necessário BAA com AWS antes de processar PHI. Solicite via AWS Artifact ou contato com suporte.

REQUISITOS: Criptografia de PHI em repouso e trânsito, logs de auditoria de acessos, controles de acesso baseados em função, backup de PHI, plano de disaster recovery, treinamento de equipe.

SERVIÇOS ELEGÍVEIS: Nem todos os serviços AWS são HIPAA eligible. Verifique lista oficial - EC2, S3, RDS são elegíveis. Serviços como Amazon Chime, WorkMail não são.

SOC 2 (Service Organization Control)

O QUE É: Framework de auditoria desenvolvido pela AICPA para avaliar controles de segurança, disponibilidade, confidencialidade de serviços.

TIPOS: SOC 2 Type I (design de controles em momento específico), Type II (eficácia de controles por período, tipicamente 6-12 meses).

TRUST SERVICE CRITERIA: Segurança (obrigatório), disponibilidade, integridade de processamento, confidencialidade, privacidade (opcionais conforme aplicável).

PARA CLIENTES AWS: AWS tem SOC 2 próprio. Você precisa SOC 2 para suas aplicações/serviços. Implemente controles documentados, evidências de operação, monitoramento contínuo, resposta a incidentes.

ISO 27001

O QUE É: Padrão internacional para sistemas de gestão de segurança da informação (ISMS). Certificação reconhecida globalmente.

ABORDAGEM: Baseada em risco - identifique riscos, implemente controles, monitore eficácia, melhore continuamente. Não prescreve controles específicos, permite flexibilidade.

CERTIFICAÇÃO: Requer auditoria externa por organismo certificador credenciado. Válida por 3 anos com auditorias anuais de supervisão.

IMPLEMENTAÇÃO: Defina escopo do ISMS, faça assessment de riscos, implemente controles do Anexo A conforme aplicável, documente políticas e procedimentos, treine equipe, monitore KPIs.

NIST Cybersecurity Framework

O QUE É: Framework voluntário desenvolvido pelo National Institute of Standards and Technology (EUA) para gerenciar riscos de cybersecurity.

5 FUNÇÕES CORE: Identify (ativos, riscos), Protect (implementar salvaguardas), Detect (atividades anômalas), Respond (ações durante incidente), Recover (restaurar serviços).

TIERS: Parcial, Risk Informed, Repetível, Adaptativo. Permite avaliar maturidade de cybersecurity da organização.

APLICAÇÃO AWS: Use como framework para construir programa de segurança. Mapeia bem para serviços AWS - GuardDuty para Detect, Security Hub para Identify, AWS Backup para Recover.

🔧 Ferramentas AWS para Conformidade

AWS Artifact

Portal central para acesso sob demanda a relatórios de conformidade e acordos da AWS. Disponível sem custo no console AWS.

RELATÓRIOS DISPONÍVEIS: SOC 1, 2, 3, ISO 27001, 27017, 27018, 27701, PCI-DSS AOC, FedRAMP, HIPAA BAA, GDPR DPA, e muitos outros.

COMO USAR: AWS Console → Artifact → Reports ou Agreements. Download de PDFs para auditorias, due diligence de clientes, assessments internos.

ACORDOS: BAA para HIPAA, DPA para GDPR podem ser aceitos diretamente via Artifact sem contato com AWS.

AWS Audit Manager

Automatiza coleta de evidências de conformidade. Reduz esforço manual de preparação para auditorias.

FRAMEWORKS PRÉ-CONSTRUÍDOS: CIS AWS Foundations Benchmark, PCI-DSS, GDPR, HIPAA, SOC 2, ISO 27001 e outros. Mapeia controles para configurações AWS.

FUNCIONAMENTO: Cria assessments baseados em framework escolhido. Coleta evidências automaticamente de CloudTrail, Config, Security Hub. Organiza evidências por controle.

RELATÓRIOS: Gera assessment reports prontos para auditores. Reduz tempo de preparação de semanas para horas.

AWS Config

Avalia, audita e analisa configurações de recursos AWS. Fundamental para conformidade contínua.

CONFIG RULES: Regras gerenciadas pela AWS ou customizadas. Avaliam se recursos estão em conformidade. Exemplos: s3-bucket-public-read-prohibited, iam-password-policy, encrypted-volumes.

REMEDIAÇÃO: Automática via Systems Manager Automation ou manual. Corrige não-conformidades sem intervenção humana quando possível.

CONFORMITY PACKS: Coleções de Config Rules e remediações. Templates para frameworks completos (CIS, PCI-DSS) em um único deployment.

AWS Security Hub

Agregação centralizada de descobertas de segurança. Dashboard único para postura de segurança.

SECURITY STANDARDS: CIS AWS Foundations Benchmark, AWS Foundational Security Best Practices, PCI-DSS. Verifica automaticamente conformidade contra standards.

INTEGRAÇÕES: GuardDuty, Inspector, Macie, IAM Access Analyzer, Firewall Manager, Config, third-party tools (Palo Alto, Trend Micro, etc).

SECURITY SCORE: Pontuação de segurança baseada em checks passados vs falhados. Facilita tracking de progresso ao longo do tempo.

AWS Control Tower

Orquestra setup e governance de ambiente AWS multi-conta seguro. Implementa landing zone com best practices.

GUARDRAILS: Preventivos (SCPs) e detectivos (Config Rules). Garantem conformidade contínua. Exemplos: proibir mudanças em CloudTrail, requerer MFA para root account.

ACCOUNT FACTORY: Provisiona contas novas automaticamente com baseline de segurança. Garante consistência em todas as contas.

DASHBOARD: Visibilidade de status de conformidade em toda organização. Identifica contas ou guardrails violados rapidamente.

AWS Systems Manager Compliance

Escaneia instâncias EC2 para conformidade com patch baselines, configurações desejadas, custom compliance types.

PATCH COMPLIANCE: Verifica se instâncias têm patches críticos e de segurança instalados. Identifica instâncias out-of-compliance.

ASSOCIATION COMPLIANCE: Valida se State Manager associations foram executadas com sucesso. Garante configurações aplicadas.

CUSTOM COMPLIANCE: Permite definir tipos customizados de compliance. Útil para requisitos específicos da organização ou indústria.

📊 Implementando Programa de Conformidade

Fase 1: Assessment e Scoping

IDENTIFIQUE REQUISITOS: Determine quais frameworks e regulamentações aplicam-se à sua organização. Considere indústria, localização geográfica, tipo de dados processados.

DEFINA ESCOPO: Quais contas AWS, regiões, serviços estão no escopo? Documentação clara de escopo é crítica para auditorias.

GAP ANALYSIS: Compare estado atual vs requisitos. Use Security Hub e Config para assessment automatizado. Identifique gaps e priorize por risco.

STAKEHOLDERS: Envolva liderança executiva, legal, compliance, segurança, operações. Conformidade é responsabilidade de toda organização.

Fase 2: Design e Planejamento

POLÍTICAS E PROCEDIMENTOS: Documente políticas de segurança, procedimentos operacionais, responsabilidades. Base para auditorias.

ARQUITETURA DE REFERÊNCIA: Desenhe arquitetura que atende requisitos. Multi-conta com segregação por ambiente e sensibilidade de dados.

CONTROLES TÉCNICOS: Liste controles específicos a implementar (criptografia, logs, access controls, network segmentation, backup, etc).

ROADMAP: Crie plano faseado de implementação. Priorize controles críticos e quick wins primeiro. Estabeleça timelines realistas.

Fase 3: Implementação

AUTOMAÇÃO: Use Infrastructure as Code (CloudFormation, Terraform) para garantir consistência. Codifique controles de segurança em templates.

CONTROLES PREVENTIVOS: SCPs, IAM policies, Security Groups, WAF rules. Impedem ações não-conformes antes que ocorram.

CONTROLES DETECTIVOS: Config Rules, GuardDuty, Security Hub, CloudWatch Alarms. Identificam violações rapidamente.

REMEDIAÇÃO: Automatize correção de não-conformidades quando possível. Config remediation actions, Lambda functions, Systems Manager Automation.

Fase 4: Operação e Monitoramento

MONITORAMENTO CONTÍNUO: Security Hub e Config fornecem visão em tempo real de postura de conformidade. Configure dashboards para stakeholders.

ALERTAS: Notificações imediatas de violações críticas via SNS, email, Slack, PagerDuty. Response rápida essencial para conformidade.

MÉTRICAS: Track KPIs - compliance score, tempo para remediação, número de violações, cobertura de controles. Demonstre melhoria ao longo do tempo.

REVISÕES REGULARES: Assessments mensais de conformidade. Revise mudanças em requisitos regulatórios. Ajuste controles conforme necessário.

Fase 5: Auditoria e Certificação

PREPARAÇÃO: Use Audit Manager para coletar evidências continuamente. Organize documentação, políticas, logs, screenshots.

AUDITORIA: Trabalhe com auditores externos (para ISO, SOC 2) ou assessors internos. Forneça acesso apropriado via IAM roles temporárias.

REMEDIAÇÃO DE FINDINGS: Corrija issues identificados por auditores prontamente. Documente planos de ação corretiva para items que requerem tempo.

CERTIFICAÇÃO: Obtenha certificados/relatórios. Compartilhe com stakeholders e clientes conforme apropriado. Mantenha vigência com recertificações.

🎯 Controles Técnicos Essenciais

Controles de Identidade e Acesso

MFA OBRIGATÓRIO: Para todos os usuários humanos, especialmente privilegiados. Requisito em praticamente todos os frameworks.

PRINCÍPIO DO MENOR PRIVILÉGIO: Permissões granulares baseadas em função. Revise e reduza permissões regularmente com Access Analyzer.

SENHA FORTE: Política IAM com mínimo 14 caracteres, complexidade, rotação regular. Previne brute force e credential stuffing.

FEDERATED ACCESS: Integração com IdP corporativo (Okta, Azure AD) via SAML/OIDC. Centraliza gestão de identidades e facilita compliance.

ACCESS REVIEWS: Revisões trimestrais de permissões. Remova usuários inativos, reduza privilégios excessivos. Documente revisões para auditoria.

Controles de Logging e Monitoramento

CLOUDTRAIL GLOBAL: Ativo em todas as regiões, criptografado com KMS, validação de integridade habilitada. Registra todas as ações API.

VPC FLOW LOGS: Logs de tráfego de rede. Essencial para investigações de segurança, detecção de anomalias, forensics.

RETENÇÃO: Mínimo 1 ano para a maioria dos frameworks. 7 anos para alguns regulamentações financeiras. Use S3 com lifecycle policies.

PROTEÇÃO DE LOGS: S3 bucket com versionamento, MFA delete, políticas de acesso restritivas. Logs não podem ser modificados ou deletados por usuários normais.

ALERTAS: CloudWatch alarms para eventos críticos - falhas de MFA, mudanças em Security Groups, desativação de logs, uso de root account.

Controles de Criptografia

DADOS EM REPOUSO: S3 SSE obrigatório, EBS encryption por padrão, RDS encryption habilitada na criação. Use KMS CMKs para controle total.

DADOS EM TRÂNSITO: TLS 1.2+ para toda comunicação. Desabilite protocolos antigos. Certificados SSL válidos e renovados automaticamente.

KEY MANAGEMENT: Rotação automática de KMS keys anual. Políticas de key restritivas. Separação de keys por ambiente (dev, prod) e sensibilidade.

SECRETS: Nunca hardcode credenciais. Use Secrets Manager ou Parameter Store. Rotação automática quando suportado pelo serviço.

VALIDAÇÃO: Use AWS Config rules para garantir encryption enforcement. Bloque criação de recursos não-criptografados via SCPs.

Controles de Rede

VPC ISOLAMENTO: Recursos sensíveis em subnets privadas. Segmentação por ambiente e tier (web, app, database).

SECURITY GROUPS: Princípio do menor acesso. Nunca 0.0.0.0/0 em portas sensíveis. Documente e revise regras regularmente.

NACLs: Camada adicional de defesa. Regras de negação explícitas para tráfego malicioso conhecido. Stateless para controle granular.

PRIVATE ENDPOINTS: VPC Endpoints para serviços AWS. Tráfego não sai da rede AWS. Reduz exposição e melhora performance.

WAF: Proteção contra OWASP Top 10. Regras gerenciadas + custom rules. Rate limiting, geo-blocking conforme requisitos.

Controles de Backup e Disaster Recovery

BACKUPS AUTOMATIZADOS: AWS Backup com políticas centralizadas. Frequência baseada em RPO (Recovery Point Objective) do serviço.

RETENÇÃO: Conforme requisitos regulatórios. Backups imutáveis quando necessário (Vault Lock). Teste de restore obrigatório.

CROSS-REGION: Cópias de backup em região secundária. Proteção contra desastres regionais. Critical para DR e algumas regulamentações.

DR PLAN: Documentado, testado regularmente. RTO (Recovery Time Objective) e RPO definidos. Runbooks para diferentes cenários.

VALIDAÇÃO: Testes de restore trimestrais. Valide que backups funcionam e atendem RTOs/RPOs. Documente resultados.

📝 Documentação e Evidências

Políticas e Procedimentos

POLÍTICA DE SEGURANÇA: Documento de alto nível aprovado pela liderança. Define compromisso organizacional com segurança e conformidade.

PROCEDIMENTOS OPERACIONAIS: SOPs (Standard Operating Procedures) detalhados. Como executar tarefas específicas mantendo conformidade.

MATRIZ DE CONTROLES: Mapeia requisitos regulatórios para controles implementados e evidências. Facilita auditorias tremendamente.

PLANO DE RESPOSTA: Incident response plan documentado. Roles, responsabilidades, procedimentos passo-a-passo, contatos de emergência.

VERSIONAMENTO: Controle de versão de todos os documentos. Data de criação, revisões, aprovações. Git ou sistema de documentação formal.

Evidências de Controles

SCREENSHOTS: Configurações de segurança implementadas. Security Hub scores, Config compliance status, IAM policies, Security Groups.

LOGS: CloudTrail logs demonstrando ações executadas. Logs de acesso. Logs de mudanças de configuração. Retenção conforme requerido.

RELATÓRIOS: Trusted Advisor checks, Config compliance reports, vulnerability scan results, penetration test reports.

CERTIFICADOS: SSL certificates, training certifications da equipe, attestations de terceiros, relatórios de auditoria.

ORGANIZAÇÃO: Sistema organizado para armazenar evidências. Por controle, por período, fácil recuperação. Audit Manager ajuda tremendamente.

Registros de Mudanças

CHANGE MANAGEMENT: Processo formal para mudanças em produção. Request, approval, testing, implementation, verification.

CONFIG CHANGES: AWS Config registra automaticamente mudanças em recursos. Timeline de configurações. Quem mudou o quê e quando.

APROVAÇÕES: Evidências de aprovações de mudanças. Emails, tickets, sistemas de workflow. Critical para demonstrar governance.

ROLLBACK CAPABILITY: Capacidade de reverter mudanças problemáticas. Procedures documentados. Testados regularmente.

POST-IMPLEMENTATION REVIEW: Validação que mudança funcionou como esperado. Sem impactos adversos em segurança ou conformidade.

Treinamento e Awareness

TREINAMENTO INICIAL: Onboarding de novos funcionários inclui treinamento de segurança e conformidade. Documentação de conclusão.

TREINAMENTO CONTÍNUO: Anual ou mais frequente. Atualizações sobre novas ameaças, mudanças regulatórias, novas ferramentas/processos.

ROLE-SPECIFIC: Treinamento adicional para roles privilegiadas. Administradores, desenvolvedores, pessoal de segurança.

REGISTROS: Mantenha registros de quem completou qual treinamento e quando. Sistema de tracking. Evidência para auditores.

TESTES: Quizzes ou assessments para validar compreensão. Phishing simulations. Demonstra eficácia do programa.

⚠️ Desafios Comuns e Soluções

Desafio: Complexidade de Múltiplos Frameworks

PROBLEMA: Organização precisa atender GDPR, PCI-DSS, ISO 27001 simultaneamente. Requisitos overlapping mas não idênticos.

SOLUÇÃO: Implemente framework mais rigoroso como baseline (geralmente ISO 27001 ou CIS). Maioria dos controles atenderá múltiplos frameworks. Identifique gaps específicos e implemente controles adicionais conforme necessário. Use matriz de controles para mapear um controle para múltiplos requisitos.

Desafio: Recursos Limitados

PROBLEMA: Equipe pequena, budget limitado. Conformidade parece overwhelming e cara.

SOLUÇÃO: Priorize automação - Security Hub, Config, Audit Manager fazem trabalho pesado. Use managed services quando possível - RDS, não EC2 com database gerenciado manualmente. Foque em controles críticos primeiro. Use frameworks da AWS (Well-Architected, Landing Zone) em vez de reinventar. Considere consultores para setup inicial, depois mantenha internamente.

Desafio: Mudanças Constantes na Nuvem

PROBLEMA: DevOps deployment contínuo. Recursos criados/modificados rapidamente. Difícil manter conformidade.

SOLUÇÃO: Shift-left security - integre compliance checks no CI/CD pipeline. Use CloudFormation Guard para validar templates antes de deploy. SCPs previnem ações não-conformes. Config detecta e remedeia automaticamente. Implemente infrastructure as code - codifique controles de segurança nos templates.

Desafio: Shadow IT

PROBLEMA: Equipes criam recursos AWS fora de processos aprovados. Descobrem-se recursos não-conformes em auditorias.

SOLUÇÃO: AWS Organizations com SCPs para enforçar guardrails. Service Catalog para provisão self-service de recursos pré-aprovados. Educação sobre riscos. Simplifique processos legítimos para reduzir tentação de shadow IT. Detecção via Config e Security Hub. Tag policies para forçar ownership.

Desafio: Evidências de Conformidade Contínua

PROBLEMA: Auditores querem evidências de conformidade ao longo do tempo, não apenas snapshot.

SOLUÇÃO: AWS Audit Manager coleta evidências continuamente. Config compliance history mostra conformidade over time. CloudTrail para audit trail completo. Security Hub scores históricos. Dashboards automatizados em QuickSight ou Grafana. Relatórios mensais de compliance gerados automaticamente.

Desafio: Custos de Conformidade

PROBLEMA: Logging extensivo, multi-região, backups, ferramentas de segurança aumentam custos significativamente.

SOLUÇÃO: Custos de não-conformidade são muito maiores (multas, perda de clientes, reputação). Otimize sem comprometer segurança - lifecycle policies para logs (transition para Glacier), retenção baseada em requisitos reais, não "forever". Use tiers gratuitos de serviços (GuardDuty 30 dias trial, Security Hub free tier). Calcule ROI de automação - reduz headcount necessário.

🔄 Manutenção de Conformidade

Atividades Contínuas

DIARIAMENTE: Revisar alertas de Security Hub, GuardDuty findings, Config non-compliance notifications. Investigar e remediar issues críticos imediatamente.

SEMANALMENTE: Revisar compliance dashboards, analisar tendências de violações, revisar mudanças de configuração significativas via Config, verificar status de remediações em andamento.

MENSALMENTE: Assessment formal de conformidade, revisar e atualizar documentação, testar backups e DR procedures, revisar access permissions, gerar relatórios para stakeholders, atualizar matriz de controles.

TRIMESTRALMENTE: Access reviews formais, training refreshers para equipe, testes de incident response, revisão de políticas e procedures, avaliação de novos serviços AWS para compliance, assessment de riscos.

ANUALMENTE: Auditorias externas conforme necessário, recertificações, revisão completa do programa de conformidade, atualização de DR plan, renovação de BAAs e DPAs, treinamento formal completo da equipe.
